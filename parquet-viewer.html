Frequently Asked Questions

What file formats are supported?

DevPalette supports both Apache Parquet (.parquet) and Apache Arrow (.arrow, .feather) file formats with automatic format detection and optimized rendering for each format type.

How large files can the viewer handle?

The Rust-based ParquetViewer backend efficiently handles large files with memory-optimized loading. Performance depends on available system memory, but files in the gigabytes range are typically well-supported.

What happened to the SQL editor feature?

DevPalette removed the SQL editor to focus on core file viewing functionality. The simplified architecture using unified ParquetViewer API provides better performance and reliability for data exploration.

Can I export data to other formats?

Currently, DevPalette exports data to JSON format directly from the ParquetViewer API. This JSON output can be easily imported into other data analysis tools and processing workflows.

Does it support nested and complex data types?

Yes! DevPalette properly displays nested structures, arrays, maps, and other complex Parquet and Arrow data types with hierarchical visualization and detailed schema inspection.

How does the unified API improve performance?

The single Rust-based ParquetViewer backend eliminates multiple dependency layers, reduces memory overhead, and provides direct JSON output, resulting in faster loading times and improved stability.
